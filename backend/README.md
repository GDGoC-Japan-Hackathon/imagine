# Imagine Backend

このディレクトリには、プロジェクトで利用されるバックエンドサーバー、および初期開発時の検証用モジュールが含まれています。

## 概要

現在の本番環境（AAOS端末向けの構成）で稼働しているのは **リレーサーバー (Node.js)** のみです。
開発当初に設計・実装された **Python バックエンド** モジュール群は、現在フロントエンド (Dart/Flutter) 側へ役割が移行されており、開発履歴および処理ロジックの参照資料として本構成に残されています。

---

### 1. ネットワークリレーサーバー (`relay.js`)

`edge/streamer_app` (スマートフォンのカメラ中継アプリ) と、`frontend/imagine` (AAOSダッシュボードアプリ) 間において、カメラ映像（バイナリ）やコマンドをリアルタイムに転送するための WebSocket 中継サーバーです。

- **役割**: スマートフォンで取得したカメラのストリーム映像を、AAOS端末上のダッシュボードアプリへと低遅延で中継します。
- **起動方法**:
  ```bash
  # Node.js環境が必要です
  node relay.js
  ```
- **ポート**: デフォルトで `8080` ポートにて WebSocket 接続を待ち受けます。

---

### 2. Python バックエンド (技術検証用 / 現行アーキテクチャでは非稼働)

ディレクトリ内に配置されている `.py` ファイル群（`main.py`, `facetrack.py`, `analyze_and_mask_object.py` など）は、プロジェクトの初期開発段階における技術検証（PoC）のために作成されたモジュール群です。

開発初期においては、カメラからの顔検出や視線検知 (MediaPipe)、画像のマスク処理といった比較的重い画像処理をバックエンドの Python で集中的に行うアーキテクチャを想定していました。
しかしその後の開発プロセスにおいて、通信によるレイテンシーの削減とシステム全体の最適化を図るため、**現在これらの機能はすべてフロントエンド側（Flutter/Dartのネイティブ実装）に移行され、エッジデバイス（AAOS端末）内で直接・高速に処理されるように進化しています。**

そのため、これらの Python スクリプト群は現在稼働している実機アプリケーションやデモでは使用しておりません。本プロジェクトが初期段階においてどのように顔検出や AI 解析の技術検証を行い、現在の「エッジデバイス上でのリアルタイム完結型アーキテクチャ」に至ったかを示す、開発プロセスの軌跡として本ディレクトリに残しています。

## ファイル構成 (参考)

- `relay.js` : （稼働中）中継用の WebSocket リレーサーバー
- `main.py`, `camera.py`, `incamera.py`, `outcamera.py` : 各種カメラ入力のハンドリング処理 (Python初期版)
- `facetrack.py` : 顔・視線トラッキング処理モジュールのプロトタイプ
- `analyze_and_mask_object.py` : 解析およびオブジェクトのマスク処理モジュールのプロトタイプ
- `model/`, `Convert/` : 初期段階で利用していた学習モデル・Dartコードへの変換処理などのリソース置き場
